# 关于Controller、Service、Dao分层由来的简易阐述
- 远古时代没有分层概念，只有一层。
- 后来发现，很多底层操作数据库的函数可以复用，于是分两层，controller和dao（model），contoller负责接http请求，dao负责包装数据库函数，controller调用dao的函数。至今，php和python很多web项目就两层。
- 再后来，需要对业务进行测试，但每次测试，都需要修改数据库，才能看到结果，太麻烦了。后来人们想到一种办法，在controller和dao之间再弄一个service层，这个service层专门对dao的方法进行抽象，service层全是接口。这样，在做单元测试的时候，可以在service层定义测试类，实现接口，这样就很容易mock数据，测试逻辑函数。
- 至于其余的层，看你实际需求，甚至service层都看需求，因为很多小项目不要求单元测试的。
